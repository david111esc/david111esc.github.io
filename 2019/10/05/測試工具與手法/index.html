軟體品質維護-測試簡介

Author: Hilbert SYU Date: October 5, 2019  14:16:39 Category: 軟體工程

前言

此篇文章主要針對測試的相關名詞進行簡介，主要是從內部訓練課程中聽到的名詞進行搜尋， 後來才發現Version Control感覺和測試沒有太直接的相關性，不過已經寫下去了，所以就納入其中，希望未來有時間再針對git的相關的功能進行深入探討。

進行測試的原因

對程式進行測試是為了維持其品質，藉由在各項測試中抓出Bug，避免最終程式上線運行時的錯誤導致更嚴重的損失。

測試的相關名詞

  1. Version Control
  2. Black Box testing
  3. White Box testing
  4. SCA(static Code Analyzer)
  5. Unit test
  6. TDD(Test-Driven Development)
  7. BDD(Behavior-Driven Development)

1. Version Control

版本控制，透過控制程式碼、文件等檔案的版本內容，確保專案中協作人員能夠同步使用到相同版本的檔案。目前常見的版本控制軟體，如:Git、Subversion(SVN)。版本控制的優點，追蹤版本改動、版本回溯、分支控管等。

2. Black Box testing

黑箱測試，黑箱的意思就是測試者無須了解程式內部運作的情況，僅針對輸入、輸出和系統功能的運作進行檢驗。測試的案例通常依照程式設計的功能需求與規範，進行有效與無效的輸入，並驗證對應的輸出是否符合預期結果。
此種測試方法適合絕大部分的軟體測試，例如:整合測試(integration testing)、系統測試(system testing)。

3. White Box testing

白箱測試，需要針對程式本身的結構、邏輯等運作進行檢驗，又稱透明盒測試(glass box testing))、結構測試(structural testing)。此種方式的測試需要充分理解程式內部的演算法、架構等資訊，然後對程式運作時資料的流動路徑、邏輯運算、系統互動等進行測試。白箱測試可以應用於單元測試、整合測試、系統軟體測試等。
白箱測試的優點在於，掌握原始碼的細節，能夠針對程式碼進行最佳化，並消除可能存在的隱藏缺陷，透過白箱測試也能提出反饋，提升軟體開發工程師未來開發的能力。缺點則是此種測試方法非常複雜且耗時，測試人員需要有足夠的技術能力才能夠進行有效的測試，另外，儘管白箱測試進行大量的檢驗，仍無法百分之百保證涵蓋所有情況，只能盡可能的增加測試的覆蓋率。

4. SCA(static Code Analyzer)

靜態程式分析意思就是不需要執行程式碼進行程式分析的方法，此種方式通常會對不同程式語言有依賴性。
SCA的工具通常能讓使用者設定不同條件，藉此檢視程式碼是否符合這些規則，SCA方法被使用的場景:

  1. Code Syle: 當軟體開發Team中有針對程式碼風格進行規範時(e.g.排版、變數命名…etc)，就能將這些規則建立在SCA工具中進行檢測，降低Code Review成本。
  2. Refactor: 重構是為了消除程式碼中的”壞味道”，消除為了追求時限內產出而累積的技術債，或者增加未來程式開發拓展的靈活性等，而所謂的”壞味道”多數也都能夠具體化的描述成規則，因此，就能利用SCA工具將這些問題找出來，加速重構程式法的效率。
  3. Debug: 許多程式碼的Bug不需要實際執行就能夠被發現。比較簡單的，例如:變數未被宣告、初始化、使用…;比較複雜的，例如:記憶體使用後未被釋出。

5. Unit test

單元測試是針對軟體設計中最小單位進行正確性檢驗的行為，程式中的單元可以是程式、函式(Function)、過程(Process)等，對物件(Object)而言最小單元就是物件的方法(Methods)。
基本上，程式每被修改一次就應該要進行單元測試，單元測試的優點是能夠對程式的細節進行檢視，缺點則是當測試的覆蓋率越高時，需要耗費大量的時間與人力成本。
單元測試通常為程式編寫者自行撰寫，好處是編寫者對於自己寫的東西較為清楚，測試程式能寫得較快速，但這樣相對的在開發上需要花費更多時間，而且有些盲點自己無法發現檢測出來。

6. TDD(Test-Driven Development)

測試驅動開發是一種軟體開發過程的方法，提倡先根據需求撰寫測試程序，再依據測試開發出符合其需求的程式。TDD優點是避免過度設計帶來的浪費，同時開發者必須真正完全了解需求，才能夠撰寫出完整的測試。缺點則是開發者可能只為了滿足測試程序而忽略對實際需求的實現(可能導致重構成本增加)。

7. BDD(Behavior-Driven Development)

行為驅動開發，對此方法還不是很清楚其定義，暫且先不介紹僅留下名詞供日後查詢。
