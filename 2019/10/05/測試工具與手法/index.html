測試工具與手法

Author: Hilbert SYU Date: October 5, 2019  14:16:39 Category: 軟體工程

進行測試的原因

對程式進行測試是為了維持其品質，藉由在各項測試中抓出Bug，避免最終程式上線運行時的錯誤導致更嚴重的損失。

測試的相關名詞

  1. SCA(static Code Analyzer)
  2. Unit test
  3. Black Box test
  4. White Box test
  5. Version Control
  6. TDD(Test-Driven Development)
  7. BDD(Behavior-Driven development)

1. SCA(static Code Analyzer)

靜態程式分析意思就是不需要執行程式碼進行程式分析的方法，此種方式通常會對不同程式語言有依賴性。
SCA的工具通常能讓使用者設定不同條件，藉此檢視程式碼是否符合這些規則，SCA方法被使用的場景:

  1. Code Syle: 當軟體開發Team中有針對程式碼風格進行規範時(e.g.排版、變數命名…etc)，就能將這些規則建立在SCA工具中進行檢測，降低Code Review成本。
  2. Refactor: 重構是為了消除程式碼中的”壞味道”，消除為了追求時限內產出而累積的技術債，或者增加未來程式開發拓展的靈活性等，而所謂的”壞味道”多數也都能夠具體化的描述成規則，因此，就能利用SCA工具將這些問題找出來，加速重構程式法的效率。
  3. Debug: 許多程式碼的Bug不需要實際執行就能夠被發現。比較簡單的，例如:變數未被宣告、初始化、使用…;比較複雜的，例如:記憶體使用後未被釋出。
